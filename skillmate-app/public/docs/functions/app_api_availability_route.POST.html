      User availability management endpoint

      Manages the authenticated user's availability time slots for scheduling sessions.

      Parameters

      • req: Request

        Request object containing availability data

        • startUtc

          Start time in UTC ISO format

        • endUtc

          End time in UTC ISO format

        • tz

          Timezone identifier (optional, defaults to 'UTC')

      Returns Promise<NextResponse<{ error: string }> | NextResponse<{ ok: boolean }>>

      JSON response with success status or error message

      Route

      POST /api/availability

      Throws

      When user is not authenticated

      Throws

      When provided times are invalid or end time is before start time

      Throws

      When availability creation fails due to database error

      Example

      const response = await fetch('/api/availability', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({
          startUtc: '2024-01-15T09:00:00.000Z',
          endUtc: '2024-01-15T17:00:00.000Z',
          tz: 'America/New_York'
        })
      });
